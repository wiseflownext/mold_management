## 五、数据库设计（MySQL）

### 5.1 ER 关系

```
users ──────────────── 1:N ──── usage_records
  │                              │
  │                              │ N:1
  │                              ▼
  │                            molds ──── 1:N ──── mold_products
  │                              │
  │                              │ 1:N
  │                              ▼
  │                      maintenance_records
  │
  └── N:1 ──── workshops

notifications ──── N:1 ──── users
                   N:1 ──── molds

reminder_settings （按模具类型配置，全局）

certification_logs ──── N:1 ──── molds（设计寿命变更日志）
```

### 5.2 表结构

#### users（员工账号）

```sql
CREATE TABLE users (
  id          BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  username    VARCHAR(50)  NOT NULL UNIQUE,
  name        VARCHAR(50)  NOT NULL,
  phone       VARCHAR(20)  DEFAULT NULL UNIQUE,
  password    VARCHAR(255) NOT NULL COMMENT 'bcrypt加密',
  role        ENUM('admin','operator') NOT NULL DEFAULT 'operator',
  workshop_id BIGINT UNSIGNED DEFAULT NULL,
  last_login  DATETIME     DEFAULT NULL,
  created_at  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (workshop_id) REFERENCES workshops(id) ON DELETE SET NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

#### workshops（车间）

```sql
CREATE TABLE workshops (
  id         BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  name       VARCHAR(50) NOT NULL UNIQUE,
  is_default TINYINT(1)  NOT NULL DEFAULT 0 COMMENT '预设车间不可删除',
  created_at DATETIME    NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME    NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

#### molds（模具）

```sql
CREATE TABLE molds (
  id                BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  mold_number       VARCHAR(50)  NOT NULL UNIQUE,
  type              ENUM('模压模具','口型模具','接角模具') NOT NULL,
  workshop_id       BIGINT UNSIGNED DEFAULT NULL,
  first_use_date    DATE         DEFAULT NULL,
  design_life       INT UNSIGNED NOT NULL COMMENT '设计寿命（次）',
  maintenance_cycle INT UNSIGNED NOT NULL COMMENT '保养周期（次）',
  usage_count       INT UNSIGNED NOT NULL DEFAULT 0 COMMENT '累计使用次数',
  status            ENUM('在用','维修中','停用','报废') NOT NULL DEFAULT '在用',
  created_at        DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at        DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (workshop_id) REFERENCES workshops(id) ON DELETE SET NULL,
  INDEX idx_status (status),
  INDEX idx_type (type),
  INDEX idx_mold_number (mold_number)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

#### mold_products（关联产品）

```sql
CREATE TABLE mold_products (
  id          BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  mold_id     BIGINT UNSIGNED NOT NULL,
  customer    VARCHAR(100) DEFAULT NULL,
  model       VARCHAR(50)  DEFAULT NULL COMMENT '车型',
  name        VARCHAR(200) NOT NULL COMMENT '产品名称',
  part_number VARCHAR(50)  DEFAULT NULL COMMENT '零件号',
  created_at  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (mold_id) REFERENCES molds(id) ON DELETE CASCADE,
  INDEX idx_mold_id (mold_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

#### usage_records（使用记录）

```sql
CREATE TABLE usage_records (
  id          BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  mold_id     BIGINT UNSIGNED NOT NULL,
  product     VARCHAR(200) NOT NULL COMMENT '产品名称',
  quantity    INT UNSIGNED NOT NULL COMMENT '生产数量（次）',
  shift       ENUM('早班','中班','晚班') NOT NULL,
  record_date DATE         NOT NULL,
  operator_id BIGINT UNSIGNED NOT NULL,
  note        TEXT         DEFAULT NULL,
  created_at  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (mold_id) REFERENCES molds(id) ON DELETE CASCADE,
  FOREIGN KEY (operator_id) REFERENCES users(id),
  INDEX idx_mold_id (mold_id),
  INDEX idx_operator_id (operator_id),
  INDEX idx_record_date (record_date)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

#### maintenance_records（维保记录）

```sql
CREATE TABLE maintenance_records (
  id          BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  mold_id     BIGINT UNSIGNED NOT NULL,
  type        ENUM('保养','维修') NOT NULL,
  content     TEXT         NOT NULL,
  record_date DATE         NOT NULL,
  operator_id BIGINT UNSIGNED NOT NULL,
  created_at  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (mold_id) REFERENCES molds(id) ON DELETE CASCADE,
  FOREIGN KEY (operator_id) REFERENCES users(id),
  INDEX idx_mold_id (mold_id),
  INDEX idx_record_date (record_date)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

#### certification_logs（鉴定报告/设计寿命变更日志）

```sql
CREATE TABLE certification_logs (
  id               BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  mold_id          BIGINT UNSIGNED NOT NULL,
  old_design_life  INT UNSIGNED NOT NULL,
  new_design_life  INT UNSIGNED NOT NULL,
  report_url       VARCHAR(500) NOT NULL COMMENT 'OSS图片地址',
  operator_id      BIGINT UNSIGNED NOT NULL,
  created_at       DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (mold_id) REFERENCES molds(id) ON DELETE CASCADE,
  FOREIGN KEY (operator_id) REFERENCES users(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

#### notifications（通知）

```sql
CREATE TABLE notifications (
  id         BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  user_id    BIGINT UNSIGNED NOT NULL COMMENT '接收人',
  type       ENUM('maintenance_overdue','maintenance_soon','status_change','usage_record') NOT NULL,
  title      VARCHAR(100) NOT NULL,
  message    VARCHAR(500) NOT NULL,
  mold_id    BIGINT UNSIGNED DEFAULT NULL,
  is_read    TINYINT(1)   NOT NULL DEFAULT 0,
  created_at DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
  FOREIGN KEY (mold_id) REFERENCES molds(id) ON DELETE SET NULL,
  INDEX idx_user_read (user_id, is_read),
  INDEX idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

#### reminder_settings（维保提醒配置）

```sql
CREATE TABLE reminder_settings (
  id                   BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  mold_type            ENUM('模压模具','口型模具','接角模具') NOT NULL UNIQUE,
  enabled              TINYINT(1)   NOT NULL DEFAULT 1,
  remaining_threshold  INT UNSIGNED NOT NULL DEFAULT 300 COMMENT '剩余次数阈值',
  warning_percent      TINYINT UNSIGNED NOT NULL DEFAULT 80 COMMENT '预警百分比',
  overdue_percent      TINYINT UNSIGNED NOT NULL DEFAULT 100 COMMENT '逾期百分比',
  updated_at           DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 6.7 维保提醒模块

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET | `/api/reminders` | 获取维保提醒列表（含紧急度计算） | 登录 |
| GET | `/api/reminders/settings` | 获取提醒配置 | 管理员 |
| PUT | `/api/reminders/settings` | 更新提醒配置 | 管理员 |

**维保提醒计算逻辑：**

```
对每个"在用"状态的模具：
  上次保养后使用次数 = usage_count % maintenance_cycle
  剩余次数 = maintenance_cycle - 上次保养后使用次数

  if 剩余次数 < 0           → overdue（逾期）
  if 剩余次数 ≤ remaining_threshold 且 ≤ 200  → critical（紧急）
  if 剩余次数 ≤ remaining_threshold           → warning（预警）
  else                       → normal（正常）
```

### 6.10 文件上传模块

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| POST | `/api/upload/image` | 上传图片到 OSS | 管理员 |

**上传流程：**
1. 客户端 `multipart/form-data` 上传图片
2. 服务端校验文件类型（JPG/PNG/HEIC）和大小（≤10MB）
3. 生成唯一文件名：`certifications/{moldId}/{timestamp}_{random}.{ext}`
4. 上传至 OSS Bucket
5. 返回 OSS 访问 URL

## 七、认证与鉴权

### 7.1 认证流程

```
客户端                          服务端
  │                               │
  │── POST /auth/login ──────────►│
  │   { phone, password }         │
  │                               │── 校验 bcrypt(password)
  │◄── { accessToken,            │
  │      refreshToken, user } ────│
  │                               │
  │── GET /api/molds ────────────►│
  │   Authorization: Bearer {AT}  │
  │                               │── 校验 JWT 签名和过期时间
  │◄── { data: [...] } ──────────│
  │                               │
  │── POST /auth/refresh ────────►│ （accessToken 过期时）
  │   { refreshToken }            │
  │◄── { accessToken(new) } ─────│
```

### 7.2 权限守卫

```typescript
// 使用装饰器标记接口权限
@Roles('admin')
@UseGuards(JwtAuthGuard, RolesGuard)
@Post('molds')
createMold(@Body() dto: CreateMoldDto) {}
```

- `JwtAuthGuard`：校验 JWT token 有效性，注入当前用户信息
- `RolesGuard`：校验用户角色是否匹配 `@Roles()` 装饰器要求
- 无装饰器的接口 = 仅需登录即可访问

## 十二、关键业务逻辑

### 12.1 新增使用记录的副作用

```
1. 写入 usage_records 表
2. 更新 molds.usage_count += quantity
3. 重新计算该模具的维保剩余次数
4. 如触发维保阈值 → 生成 notification
```

### 12.2 修改设计寿命流程

```
1. 校验角色 = admin
2. 上传鉴定报告图片至 OSS → 获取 URL
3. 写入 certification_logs（old_design_life, new_design_life, report_url）
4. 更新 molds.design_life = new_design_life
```

### 12.3 模具删除

```
1. 校验角色 = admin
2. 级联删除：mold_products, usage_records, maintenance_records, certification_logs
3. notifications 中 mold_id 置 NULL
4. 删除 molds 记录
```

### 12.4 客户列表（动态聚合）

```sql
SELECT DISTINCT customer FROM mold_products WHERE customer IS NOT NULL ORDER BY customer;
```

用于模具列表页的客户筛选下拉框，从已有数据中动态聚合，无需单独维护客户表。
